<objective>
Refactor AnimatedTerminal to eliminate React re-render jank by using direct DOM patching.

Purpose: The current implementation calls `setCurrentFrame()` on every tick (~32fps), triggering a full React re-render of all 41 terminal rows. The fix pre-builds row divs once and patches only changed rows directly via `innerHTML`, matching the `aidx-landing.html` `AnimationManager._renderFrame()` pattern.
Output: `src/components/animated-terminal/index.tsx` rewritten to use ref-based DOM updates. Animation plays at consistent 32fps without jank.
</objective>

    Rewrite `src/components/animated-terminal/index.tsx` using the following approach:

    **Strategy**: Keep `Terminal` as the chrome (header, border, sizing, fonts). Pass `lines={[]}` and `disableScrolling={true}` to `Terminal`. Use a `useEffect` to find the content div inside the terminal and pre-populate it with row `<div>` elements, then patch them directly on each frame.

    **Implementation**:

    1. Remove `useState` for `currentFrame`. Remove the `setCurrentFrame` callback from `AnimationManager`.

    2. Add refs:
       ```tsx
       const containerRef = useRef<HTMLDivElement>(null);
       const rowDivsRef = useRef<HTMLDivElement[]>([]);
       const prevLinesRef = useRef<string[] | null>(null);
       const frameIndexRef = useRef(16);
       ```

    3. Keep `AnimationManager` class as-is (it's correct). Change its constructor callback to call a `renderFrame` function directly instead of `setCurrentFrame`.

    4. In `useEffect` (runs once on mount), after the component mounts:
       - Find the content element: `containerRef.current?.querySelector('[class*="content"]')` or use a more direct approach
       - Clear it and pre-build row divs:
         ```tsx
         const rowCount = frames[0].length;
         const divs: HTMLDivElement[] = [];
         for (let i = 0; i < rowCount; i++) {
           const d = document.createElement('div');
           contentEl.appendChild(d);
           divs.push(d);
         }
         rowDivsRef.current = divs;
         ```
       - Define `renderFrame`:
         ```tsx
         const renderFrame = (lines: string[]) => {
           const prev = prevLinesRef.current;
           const padding = ' '.repeat(whitespacePadding);
           for (let i = 0; i < lines.length; i++) {
             if (!prev || prev[i] !== lines[i]) {
               divs[i].innerHTML = `${padding}${lines[i]}${padding}`;
             }
           }
           prevLinesRef.current = lines;
         };
         ```
       - Create `AnimationManager` with callback:
         ```tsx
         const mgr = new AnimationManager(() => {
           frameIndexRef.current = (frameIndexRef.current + 1) % frames.length;
           renderFrame(frames[frameIndexRef.current]);
         }, baseFps);
         ```
       - Render initial frame: `renderFrame(frames[frameIndexRef.current]);`
       - Set up focus/blur/keyup listeners (same as current implementation)
       - Start if visible: `if (document.visibilityState === 'visible') mgr.start();`
       - Return cleanup: pause manager, remove listeners

    5. Render: `<Terminal ref={containerRef} ... lines={[]} disableScrolling={true} />`
       - `Terminal` needs to forward a ref to its root div. Check if it already does — if not, wrap with `forwardRef` or use a wrapper div.

    **Handling Terminal ref**: The current `Terminal` component does not use `forwardRef`. The simplest approach is to wrap `AnimatedTerminal`'s render in a `<div ref={containerRef}>` that wraps `<Terminal>`, then query inside it:
    ```tsx
    <div ref={containerRef}>
      <Terminal ... lines={[]} disableScrolling={true} />
    </div>
    ```
    Then find the content: `containerRef.current?.querySelector('[class*="content"]')`.

    **CSS Modules class name**: The content div has a CSS Modules class like `Terminal_content__xxxx`. Use `querySelector('[class*="content"]')` to find it without hardcoding the hash.

    **Preserve all existing behavior**:
    - Konami code easter egg (speed toggle) — keep exactly as-is
    - `window.focus`/`window.blur` → start/pause
    - `prefers-reduced-motion` check
    - `document.visibilityState === 'visible'` check before starting
    - `baseFps` calculation from `frameLengthMs`

    **whitespacePadding**: Apply padding in `renderFrame` as `' '.repeat(whitespacePadding)` prefix and suffix on each line's innerHTML, matching the current Terminal behavior.

    The final component should have NO `useState` calls (except possibly for platform detection if needed — but `AnimatedTerminal` doesn't use platform detection, that's in `Terminal`).
