**1. [Rule 3 - Blocking] Missing next-env.d.ts in worktree**
- **Found during:** Task 3 TypeScript verification
- **Issue:** `npx tsc --noEmit` reported 18 errors — all `Cannot find module '*.module.css'` and `Cannot find module '*.svg'`. The `next-env.d.ts` file (which references `next/types/global.d.ts` for CSS module declarations and `next/image-types/global.d.ts` for SVG declarations) is auto-generated by Next.js and untracked in the main repo, so it was absent from the worktree.
- **Fix:** Created `next-env.d.ts` in the worktree root with the standard Next.js content.
- **Files modified:** `next-env.d.ts`
- **Commit:** 407fc87

**2. [Rule 1 - Bug] NavTree internal function name collision with exported types**
- **Found during:** Task 1 implementation
- **Issue:** Ghostty source uses `FolderNode`, `LinkNode`, `BreakNode` as both exported type names and internal render function names. TypeScript would flag the function declarations as shadowing the type exports.
- **Fix:** Renamed internal render functions to `FolderNodeComponent`, `LinkNodeComponent`, `BreakNodeComponent` while keeping exported type names unchanged.
- **Files modified:** `src/components/nav-tree/index.tsx`
- **Commit:** 24bc8c2
